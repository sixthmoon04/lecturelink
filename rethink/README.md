# rethink — III 단계 성찰 + 토론 작업 공간

## 폴더 구조 규칙

각 과제(task)마다 하위 폴더를 생성한다.
폴더명 형식: `task_{번호}_{과제요약}`

### 하위 폴더 내 파일 생성 순서

1. `claude_reflection.md`   — Claude Code 독립 성찰 (III-1,2,3 기준)
2. `codex_reflection.md`    — Codex 독립 성찰 (Codex가 작성)
3. `claude_peer_review.md`  — Claude Code의 Codex 성찰 상호평가
4. `codex_peer_review.md`   — Codex의 Claude Code 성찰 상호평가 (Codex가 작성)
5. `claude_final.md`        — Claude Code 최종 성찰
6. `codex_final.md`         — Codex 최종 성찰 (Codex가 작성)

## III 단계 절차

1. 사용자가 최종 결과물에 A+~F 등급 부여
2. Claude Code → `claude_reflection.md` 작성 (Codex 성찰과 독립)
3. Codex → `codex_reflection.md` 작성 (Claude Code 성찰과 독립)
4. 성찰 교환 후:
   - Claude Code → `claude_peer_review.md` 작성
   - Codex → `codex_peer_review.md` 작성
5. 상호평가 교환 후:
   - Claude Code → `claude_final.md` 작성
   - Codex → `codex_final.md` 작성
6. 사용자에게 최종 성찰 '자연어 표' 형태로 보고
7. 사용자 최종 승인 → [정언 명령] 확정

## 성찰 기준 (III-1,2,3)

- III-1: 사용자 경험 — 왜 쓰는지, 어떻게 쓰는지, 무엇을 얻으려 하는지
- III-2: 불편함 재고 — 이용 과정 중 모든 마찰 요소 빠짐없이 점검
- III-3: 유인 요소 — 사용자가 이 결과물을 원하게 만들 강력한 동기 제안

## VI 단계 — 성찰 기반 결과물 수정

III 단계에서 승인된 [정언 명령]을 기반으로 결과물의 구현, 인터랙티브 기능, 비주얼, 콘텐츠 품질을 수정한다.

### 역할 분담

- Claude Code: 정언 명령과 사용자 자연어 요구를 종합해 수정사항을 찾고 구조화한다.
- Codex: Claude Code가 구조화한 수정사항을 실제 코드와 데이터에 반영한다.
- Claude Code: Codex가 수정한 결과물을 다시 검수하고 피드백한다.

I, II 단계의 역할 한정은 유지한다. VI 단계에서도 수정사항 발견과 평가의 주체는 Claude Code이고, Codex는 하부 실행자로서 수정 구현을 담당한다.

### 완료 기준

수정 완료의 기준은 "수정사항을 빠짐없이 모두 찾고, 수정한 결과물이 A+ 등급을 달성할 수 있는 상태"로 삼는다.

### VI 검수 및 수정 중심 요소

1. **강의록-기출-해설 간 정보 연결성**
   - 강의록, 기출, 해설 매핑이 정확한지 실제 홈페이지를 돌려보며 사용자 입장에서 검수한다.
   - 연결성 문제가 발견되면 정언 명령에 따라 수정한다.

2. **의학적 정보 정확성**
   - 강의록, 문제, 선지, 해설 내 의학적 서술이 서로 충돌하지 않는지 검수한다.
   - 필요 시 PubMed 등 권위 높은 의학 전문 외부 자료 2개 이상을 사용해 검수한다.

3. **인터랙티브 기능 정확성**
   - 모든 버튼, 탭, 화면 크기 변화, 줄글 리플로우 등 사용자 조작에 따른 변화가 자연스럽고 정확한지 실제 홈페이지에서 검수한다.

4. **AI 자체 제작 콘텐츠 완성도**
   - 3step, 지엽 등 AI 생성 콘텐츠는 나열식 서술을 금지하고 논리적 구조화, 요약성, 위계, 순서, 분리와 연결을 기준으로 검수한다.
   - 핵심 내용 누락, 과도한 내용 밀집, 조각어, OCR 노이즈, 메타 문장은 수정 또는 제거한다.

5. **비주얼 가독성과 심미성**
   - 학기 중 수업 직후 복습과 시험 직전 공부 모두에서 장시간 사용해도 피로도가 낮은지 확인한다.
   - 화면 구성, 색상, 이미지, 이모지, 글줄 길이, 여백, 대비가 조화롭고 자극적이지 않은지 검수한다.

위 5개 요소 외의 수정사항을 발견하면, 수정 도중 사용자 허가를 받은 뒤 진행한다.

### [정언 명령 01] — task_01 / 2026-04-26 승인
출처: [claude_final.md](task_01_2단계_기능수정_기능추가/claude_final.md)

**명령 1. I-1 가정 검증 의무화**
I-1 프롬프트 작성 전, 핵심 가정을 명시적으로 열거하고 각 가정의 현실적 성립 여부를 점검하는 단계를 반드시 거친다.

**명령 2. I-3 콘텐츠 시뮬레이션 의무화**
콘텐츠를 생성하는 기능(gen_*.py 포함)의 I-3 평가 시, 생성된 결과물 3개를 전문 열람한 후 평가를 시작한다. 열람 전 평가 완료 불가.

**명령 3. 품질 미달 데이터 노출 금지**
기술 한계는 품질 저하의 면죄부가 아니다. 품질 기준 미달 데이터는 "데이터 없음"으로 처리한다. I-3 평가 기준에 "노출 불가 데이터 0건"을 항목으로 포함한다.

**명령 4. 유인 요소 위계 준수**
유인 요소 제안 시 아래 위계 순서를 따른다. 상위 순위가 안정되지 않은 상태에서 하위 순위를 먼저 구현하지 않는다.
- 1순위: 강의록 키워드 → 해당 기출 문제 즉시 이동
- 2순위: 예측 문제 TOP 5 (콘텐츠 품질 안정 후)
- 3순위: 틀린 문제 기반 개인 약점 분석 (세션 저장 구조 전제)
- 4순위: 오늘의 야마 랜덤 문제 3개 (상위 순위 안정 후)

### [정언 명령 02] — task_001_add234_quality_rework / 2026-04-26 승인
출처: [codex_final.md](task_001_add234_quality_rework/codex_final.md)

**명령 1. 전체 학습 기간 기준 준수**
콘텐츠 생성 작업에서는 "파일 생성 완료"가 아니라 "사용자가 학기 중 수업 직후 복습과 시험 대비에 믿고 사용할 수 있음"을 완료 기준으로 삼는다.

**명령 2. 실제 사용자 시나리오 검증 의무화**
문제, 해설, 키워드, 예측 문구는 최소 1개 이상 실제 사용자 시나리오로 읽고 검증한다.

**명령 3. 생성 노이즈 실패 처리**
조각어, OCR 노이즈, 메타 문장, 자동 생성 흔적이 보이면 해당 생성 규칙을 실패로 간주한다.

**명령 4. 근거 없는 의학 지식 생성 금지**
근거 문맥 없는 신규 의학 지식 생성은 금지한다.

**명령 5. 품질 미달 데이터 노출 금지**
품질 기준 미달 데이터는 개수를 채우기 위해 노출하지 않는다.

**명령 6. Codex 품질 방어선 의무**
프롬프트가 부족해도 결과물이 사용자 신뢰를 해칠 경우 멈추고 기준 보완을 요청하거나 스스로 더 엄격한 기준을 적용한다.

**명령 7. 강의록-기출 연결 우선**
사용자가 원하는 강력한 유인은 "더 많은 자동 생성"보다 "수업 직후 복습과 시험 직전 정리를 모두 돕는 강의록-기출의 신뢰 가능한 연결"에서 우선 찾는다.
