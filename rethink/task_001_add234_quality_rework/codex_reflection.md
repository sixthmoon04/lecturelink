# Codex 독립 성찰 — ADD-2/3/4 품질 재작업 C- 등급

## 0. 등급 수용

사용자가 C- 등급을 매긴 이유는 타당하다. 나는 명세의 표면 조건, 즉 파일 재생성, 개수 출력, 길이 제한, API 반영 같은 실행 완료 여부에 과도하게 집중했다. 하지만 이 결과물의 실제 사용자는 의학 시험을 준비하는 학습자이며, 이들이 원하는 것은 "그럴듯한 JSON"이 아니라 신뢰 가능한 개념 정리, 짧지만 의학적으로 의미 있는 문제, 기출과 강의록 사이의 납득 가능한 연결이다. 그 핵심 품질을 충분히 보장하지 못했다.

## III-1. 사용자 경험 관점 성찰

사용자는 이 홈페이지를 단순 열람 도구로 쓰지 않는다. 사용 과정은 대략 다음과 같다.

1. 특정 차시를 선택한다.
2. 강의록으로 개념을 빠르게 훑는다.
3. 기출 문제로 출제 양상을 확인한다.
4. 3step 문제로 암기와 이해를 점검한다.
5. 지엽 탭에서 놓치기 쉬운 요소를 마지막으로 보강한다.

이 흐름에서 사용자가 얻고자 하는 것은 시간 절약, 출제 가능성 판단, 반복 학습의 효율, 그리고 "이 내용을 믿고 공부해도 된다"는 신뢰감이다. 그런데 내가 생성한 결과물은 다음 지점에서 이 신뢰를 깨뜨렸다.

- 개념 제목에 강의 도입 멘트나 문장 조각이 남아 학습 단위처럼 보이지 않았다.
- 키워드에 `태우는연기`, `과목이라기`, `병력이` 같은 PDF 추출 부산물이 섞였다.
- 신규 문제 오답이 실제 의학 판단을 요구하지 않고 "관련 개념이다" 수준의 문장으로 끝났다.
- 검증을 `개수`, `문자 길이`, `중복 여부` 중심으로만 수행했고, 학습자가 읽었을 때 문제로 기능하는지는 충분히 보지 않았다.

앞으로 코드 생성 과정에서는 "API가 동작하는가"보다 먼저 "사용자가 이 문장을 공부 자료로 믿을 수 있는가"를 판단 기준으로 삼아야 한다.

## III-2. 불편함과 마찰 요소 성찰

사용자가 불편함을 느낄 지점은 많았다.

- 학습 가이드의 제목이 불안정하면 사용자는 각 accordion을 열기 전에 이미 품질을 의심한다.
- 키워드가 의미 없는 조각이면 badge UI가 오히려 혼란을 만든다.
- 심화형 문제의 정답이 "핵심 개념이다" 같은 메타 문장이면 시험 대비 문제가 아니라 자동 생성 흔적으로 보인다.
- 오답이 모두 "관련 개념이다" 식이면 사용자는 선지를 비교하며 사고할 수 없다.
- 지엽 탭이 최소 개수를 채우기 위해 조각 단어를 포함하면 "비강조지만 기출된 요소"라는 탭의 약속이 무너진다.
- 나는 차시12와 차시7만 부분적으로 확인하면서도, 그 샘플에 드러난 품질 문제가 전 차시에 퍼질 수 있다는 점을 충분히 일반화하지 못했다.

수정해야 할 방향은 명확하다.

- PDF 원문 추출 기반 생성은 반드시 노이즈 제거, 용어 정규화, 샘플 품질 검사를 함께 가져야 한다.
- 숫자상 조건을 만족해도 자연어 품질이 낮으면 실패로 처리해야 한다.
- 의학 문제 생성은 원문 조각 재배열이 아니라 "질문, 정답 근거, 헷갈릴 만한 오답" 구조를 가져야 한다.
- 지엽 항목은 최소 개수보다 의미 있는 항목 우선이어야 하며, 최소 개수 채우기 fallback도 의학 용어 whitelist 또는 기출 문맥 기반 필터를 통과해야 한다.

## III-3. 사용자가 원하게 만들 유인 요소

이 결과물을 사용자가 계속 쓰고 싶게 만들려면 단순 자동 생성보다 다음 기능적 유인이 필요하다.

- 각 개념 단위에 "왜 중요한가" 한 줄을 붙여 출제 가능성과 학습 우선순위를 즉시 알게 한다.
- 3step 문제에 "정답 근거 문장"과 "오답이 틀린 이유"를 짧게 붙인다.
- 지엽 탭에는 "기출 문장 원문 발췌"와 "강의록 문맥"을 나란히 보여 사용자가 납득할 수 있게 한다.
- 자동 생성 품질 점수를 내부적으로 계산하고, 기준 미달 항목은 표시하지 않거나 "검토 필요"로 분리한다.
- 차시별로 "오늘 10분 복습 세트"처럼 바로 공부를 시작하게 하는 작은 학습 루틴을 제공한다.

## 다음 코드 생성에 적용할 Codex 원칙

1. 생성 데이터는 개수보다 품질을 우선한다.
2. 샘플 출력에서 품질 문제가 보이면 해당 문제를 국소 예외로 보지 않고 생성 규칙 전체의 결함으로 간주한다.
3. PDF 추출 텍스트를 그대로 문제나 제목으로 쓰지 않는다.
4. 키워드는 의학 용어 whitelist, 기출 등장 문맥, 불용어/조각어 필터를 모두 통과해야 한다.
5. 문제 선지는 "비교하고 판단할 수 있는 문장"이어야 하며, 메타 문장이나 자동 생성 흔적을 금지한다.
6. 최종 보고 전에 최소한 사용자가 지정한 대표 차시 외에도 문제 많은 차시, PDF 추출이 깨지는 차시, 데이터 없는 차시를 표본 검증한다.
