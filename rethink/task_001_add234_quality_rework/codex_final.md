## 최종 핵심 명제

나는 이번 작업에서 "코드와 JSON을 생성했다"는 사실을 "학습자가 학기 중 수업 직후 복습과 시험 대비 전 과정에서 믿고 사용할 수 있는 학습 콘텐츠를 만들었다"는 사실과 혼동했다.

앞으로 Codex는 형식 구현의 실행자가 아니라, 사용자 신뢰를 해칠 결과물을 막는 마지막 품질 방어선이어야 한다.

## 최종 성찰 표

| 영역 | 최종 성찰 | 이후 정언 명령으로 삼을 행동 |
|---|---|---|
| 사용자 목적 | 사용자는 이 앱을 시험 직전에만 쓰는 것이 아니라, 학기 중 수업 직후 복습부터 시험 직전 압축 정리까지 전체 기간 동안 공부 시간을 줄이고 점수를 높이기 위해 쓴다. | 모든 기능은 "수업 직후 복습과 시험 대비 모두에 실제로 도움이 되는가"를 기준으로 판단한다. |
| C-의 핵심 원인 | JSON 구조, 개수, 길이 제한은 맞췄지만 의학적으로 의미 있는 문제와 개념 정리를 만들지 못했다. | 형식 검증과 별개로 도메인 품질 검증을 반드시 수행한다. |
| 실행자 책임 | 프롬프트에 내용 품질 기준이 부족했더라도, Codex는 이상한 샘플을 보고 멈췄어야 했다. | 결과물이 사용자 신뢰를 해칠 수준이면 구현을 계속하지 말고 품질 기준을 역제안한다. |
| 검증 방식 | API 응답 확인은 사용자 경험 검증이 아니다. 사용자는 문제를 읽고, 선지를 비교하고, 답을 확인한다. | 콘텐츠 생성 후 대표 문항을 실제 사용자처럼 읽고 풀어보는 시뮬레이션을 수행한다. |
| PDF/OCR 한계 | 깨진 PDF 추출 텍스트를 그대로 제목, 키워드, 선지로 쓰면 기능이 아니라 신뢰 훼손이 된다. | 기준 미달 텍스트는 노출하지 않고, 숨기거나 "검토 필요"로 분리한다. |
| 문제 생성 | "A는 B 관련 개념이다"는 의학 시험 선지가 아니다. 오답은 헷갈릴 만한 임상 사실이어야 한다. | 선지는 질환, 병태생리, 진단, 치료, 검사, 영상, 예후, 합병증 중 하나의 판단축을 가져야 한다. |
| 근거 기반 생성 | 강의록 PDF만으로 고품질 신규 의학 지식을 생성하는 것은 위험하다. | 근거 문맥 없는 신규 의학 지식 생성은 금지한다. 기출 rawText, 해설, 강의록 문맥 중 최소 하나와 연결한다. |
| 지엽 콘텐츠 | 최소 개수를 채우기 위해 조각 단어를 넣으면 "지엽" 탭의 약속이 무너진다. | 지엽 항목은 기출 문맥과 강의록 문맥이 모두 납득 가능할 때만 노출한다. |
| 샘플링 | 차시12와 차시7 같은 대표 샘플에서 문제가 보이면 전체 생성 규칙의 결함으로 봐야 한다. | 문제 많은 차시, PDF 깨짐이 심한 차시, 데이터 없는 차시, 사용자가 지적한 차시는 필수 표본 검증군으로 삼는다. |
| 유인 요소 | 가장 가치 있는 개선은 새 장식 기능이 아니라 수업 직후에는 개념 복습을 돕고, 시험 전에는 기출 회수로 이어지게 하는 강의록 개념과 기출 문제의 직접 연결이다. | 다음 개선 우선순위는 "강의록 개념 → 관련 기출 즉시 이동"으로 둔다. |

## 정언 명령 후보

사용자가 승인하면 다음 원칙을 이후 Codex 작업에서 거스를 수 없는 기준으로 따른다.

1. 콘텐츠 생성 작업에서는 "파일 생성 완료"가 아니라 "사용자가 학기 중 복습과 시험 대비에 믿고 사용할 수 있음"을 완료 기준으로 삼는다.
2. 문제, 해설, 키워드, 예측 문구는 최소 1개 이상 실제 사용자 시나리오로 읽고 검증한다.
3. 조각어, OCR 노이즈, 메타 문장, 자동 생성 흔적이 보이면 해당 생성 규칙을 실패로 간주한다.
4. 근거 문맥 없는 신규 의학 지식 생성은 금지한다.
5. 품질 기준 미달 데이터는 개수를 채우기 위해 노출하지 않는다.
6. 프롬프트가 부족해도 결과물이 사용자 신뢰를 해칠 경우 멈추고 기준 보완을 요청하거나 스스로 더 엄격한 기준을 적용한다.
7. 사용자가 원하는 강력한 유인은 "더 많은 자동 생성"보다 "수업 직후 복습과 시험 직전 정리를 모두 돕는 강의록-기출의 신뢰 가능한 연결"에서 우선 찾는다.
