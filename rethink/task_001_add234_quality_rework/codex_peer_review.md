# Codex의 Claude Code 성찰 상호평가

작성자: Codex  
참조한 Claude Code 성찰: `rethink/task_01_2단계_기능수정_기능추가/claude_reflection.md`  
등급 맥락: C-

## 0. 전제

현재 작업 폴더 `task_001_add234_quality_rework` 안에는 `claude_reflection.md`가 없었지만, 전체 `rethink/` 검색 결과 `task_01_2단계_기능수정_기능추가/`에 Claude Code의 독립 성찰, 상호평가, 최종 성찰이 존재했다. 내용상 동일한 C- 등급과 ADD-2/3/4 품질 실패를 다루므로, 해당 Claude Code 성찰을 기준으로 Codex 상호평가를 작성한다.

## 1. 수용하는 부분

### [수용] "형식 기준과 내용 품질 기준을 분리하지 못했다"는 진단

Claude Code는 I-1 프롬프트가 5지선다, 50자 제한, JSON 구조 같은 형식 기준에는 구체적이었지만, "의학적으로 의미 있는 선지인가"라는 내용 기준을 명확히 하지 않았다고 성찰했다. 이 지적은 정확하다. Codex 역시 그 빈 공간을 스스로 메우지 못했고, 결과적으로 문장 길이는 맞지만 학습 가치가 없는 선지를 만들었다.

Codex 최종 성찰에는 "명세가 형식만 제시할 때도 도메인 맥락에서 내용 품질 기준을 추론해 적용해야 한다"는 원칙을 넣어야 한다.

### [수용] "실제로 문제를 하나 풀어봤다면 바로 발견됐을 것"이라는 지적

Claude Code의 가장 중요한 지적이다. API 구조 검증이나 JSON 개수 확인은 사용성 검증이 아니다. 사용자는 `3step` 탭에서 문제를 읽고, 선지를 비교하고, 정답을 확인한다. 이 경로를 한 번만 시뮬레이션했어도 "중환자는 중증 및 인지와 관련된 핵심 개념이다" 같은 선지가 문제로 기능하지 않는다는 사실을 즉시 알 수 있었다.

Codex도 같은 실패를 했다. 앞으로 콘텐츠 생성 기능은 "샘플 전문 출력"만으로 부족하며, 최소 1개 문항을 사람처럼 읽고 "풀 수 있는 문제인가"를 판단해야 한다.

### [수용] "기술 한계는 품질 저하의 면죄부가 아니다"

Claude Code는 OCR 노이즈와 PDF 파싱 한계를 이유로 품질 문제를 수용한 것이 오판이었다고 했다. 완전히 수용한다. PDF 추출이 불안정하면 더 많은 필터를 만들거나, 기준 미달 데이터를 숨기거나, 사용자에게 "검토 필요"로 분리해야 한다. 깨진 텍스트를 학습 콘텐츠처럼 노출하는 것은 기능 추가가 아니라 신뢰 훼손이다.

### [수용] 강의록-기출 연결이 가장 우선순위 높은 유인 요소라는 판단

Claude Code의 최종 성찰은 여러 유인 요소 중 "강의록 개념 → 해당 기출 문제 즉시 이동"을 최우선으로 골랐다. 이것은 구현 난이도 대비 가치가 크다. 이미 문제 데이터에는 `topic`, `rawText`, `session`이 있고, study guide에는 `keywords`가 있으므로 키워드 기반 연결을 만들 수 있다. 사용자는 강의록을 읽다가 "이게 어떻게 출제됐지?"를 바로 확인할 수 있다. 이는 시험 대비 앱의 반복 사용 이유가 된다.

## 2. 비판하는 부분

### [비판] Claude Code 성찰은 Codex의 실행 책임을 충분히 강하게 묻지 않았다

Claude Code는 자신이 프롬프트에 의학적 오답 기준을 명시하지 않은 점을 주로 반성했다. 그러나 Codex는 단순 실행자가 아니라 코드와 데이터를 실제로 생성한 주체다. 프롬프트가 불완전해도, 결과 샘플이 명백히 이상하면 그 자리에서 멈추고 생성 규칙을 재설계했어야 했다. "프롬프트가 부족했다"는 설명은 Codex의 품질 판단 실패를 덜어주지 않는다.

Codex 최종 성찰에는 "프롬프트가 부실한 경우에도 출력물이 사용자 신뢰를 해칠 수준이면 구현을 중단하고 품질 기준을 역제안한다"를 포함해야 한다.

### [비판] "AI가 PDF 텍스트에서 의학 시험 수준 선지를 생성할 수 없다"는 표현은 다소 절대적이다

Claude Code 최종 성찰은 PDF 텍스트에서 의학 시험 수준 선지를 자동 생성할 수 있다는 가정 자체가 틀렸다고 했다. 이 판단은 현재 결과물에 대해서는 타당하지만, 완전히 불가능하다고 결론내리기보다는 조건을 나누는 것이 더 정확하다.

- 강의록 PDF만으로는 고품질 오답 생성이 어렵다.
- 기출 rawText, 정답, 해설, 같은 차시의 실제 오답 패턴을 함께 쓰면 제한적 품질의 변형 문제는 가능하다.
- 출처 문장과 매핑되지 않는 신규 지식 생성은 금지하거나 검토 필요로 분리해야 한다.

따라서 최종 원칙은 "생성 불가"가 아니라 "근거 문맥 없는 신규 의학 지식 생성 금지"가 더 실용적이다.

### [비판] ADD-1 예측 텍스트의 빈약함은 더 강하게 다뤄야 한다

Claude Code는 ADD-1의 "2026년 출제 가능성 높음" 문장이 정보가 부족하다고 지적했지만, 최종 성찰에서는 우선순위가 낮아졌다. 그러나 사용자는 교수별 출제경향 분석을 보고 "그래서 무엇을 공부해야 하나"를 알고 싶어 한다. 예측 텍스트는 단순 추세가 아니라 구체 키워드, 최근 출제 문항, 미출제 강의 단위를 함께 보여줘야 한다. 이것도 학습 신뢰와 직접 연결된다.

## 3. Claude Code 성찰에서 Codex 최종 성찰에 반영할 내용

| 평가 항목 | Claude Code 성찰 요약 | Codex 수용/비판 | Codex 최종 성찰 반영 |
|---|---|---|---|
| 사용자 경험 이해 | 사용자는 의대 시험 대비를 위해 시간 절약과 합격 가능성 상승을 원한다 | 수용 | 기능 완료보다 학습 신뢰와 시험 대비 효용을 우선한다 |
| C- 원인 진단 | 형식 기준은 있었지만 내용 품질 기준이 없었다 | 수용 | 형식 검증과 별개로 도메인 품질 검증을 의무화한다 |
| 검증 방식 | API 구조 확인이 아니라 실제 문제 풀이 시뮬레이션이 필요했다 | 강하게 수용 | 생성 후 대표 문항을 사람이 읽는 방식으로 검증한다 |
| PDF/OCR 한계 | 노이즈 데이터는 노출하지 않는 것이 낫다 | 수용 | 기준 미달 콘텐츠는 숨기거나 검토 필요로 분리한다 |
| 유인 요소 | 강의록 개념에서 관련 기출로 즉시 이동하는 기능이 최우선이다 | 수용 | 다음 개선 우선순위로 강의록-기출 연결을 둔다 |
| AI 생성 가능성 | PDF 텍스트만으로 의학 시험 문제 생성은 비현실적이다 | 부분 수용 | 근거 문맥 없는 신규 의학 지식 생성은 금지한다 |
| Codex 책임 | 프롬프트 미비가 핵심 원인이다 | 보완 필요 | Codex도 품질 기준을 스스로 역제안해야 한다 |

## 4. 결론

Claude Code의 성찰은 C- 등급의 핵심을 대체로 정확히 짚었다. 특히 "형식 기준과 내용 품질 기준의 분리 실패", "실제 사용자 시나리오 검증 부재", "기술 한계를 품질 저하의 근거로 삼은 오판"은 Codex가 반드시 수용해야 할 지적이다.

다만 Codex는 여기에 한 가지를 더해야 한다. 실행자는 단지 프롬프트를 따르는 손이 아니라, 결과물이 사용자 신뢰를 무너뜨릴 때 멈춰야 하는 마지막 방어선이다. 다음 최종 성찰은 이 책임을 정언 명령 수준으로 명문화해야 한다.
