# Claude Code의 Codex 성찰 상호평가
작성자: Claude Code (Codex 성찰 열람 후 작성)

---

## 수용하는 부분

**[수용] 런타임 API 응답 검증 부재 지적 (III-2, 항목 4)**
Codex가 지적한 "생성된 JSON을 app.py가 런타임 검증 없이 그대로 전달한다"는 내가 놓친 부분이다. 나는 I-3에서 curl로 API 응답의 구조만 확인했고, 선지 개수나 길이 범위 검증을 코드 수준에서 강제하지 않았다. 이 지적은 타당하며 이후 프롬프트에 '서빙 전 최소 검증 로직 삽입' 기준을 추가해야 한다.

**[수용] 기출/AI 생성 문제 구분 표시 부재 (III-2, 항목 5)**
Codex의 지적이 정확하다. I-1 프롬프트에서 "[야마형 N/40]", "[심화형 N/40]" 배지를 명세했지만, 이것이 '기출인지 AI 생성인지'를 명확히 구분하는 것인지는 프롬프트에서 불분명했다. 사용자 신뢰도의 핵심 문제다.

**[수용] gen_3step.py 선지 생성 로직의 근본 결함 진단 (III-2, 항목 1)**
Codex가 `f"{kw1}는 {kw2}와 다른 개념이다"` 형태의 구조적 한계를 정확히 진단했다. 내 성찰에서는 "의학적 오답 기준을 명시하지 않았다"는 원인 수준에서 멈췄는데, Codex는 코드 로직 수준에서 더 구체적으로 짚었다. 보완적이다.

**[수용] "해설 없음" 명시적 표기 (III-3, 항목 1)**
현재 해설 없는 문제에서 [답과 해설] 버튼을 누르면 빈 공간이 나온다. 이것이 버그처럼 보일 수 있다. "해설 없음" 명시는 신뢰를 오히려 높이는 UI 처리다. 수용한다.

## 비판하는 부분

**[비판] III-1에서 사용자 경험 분석이 추상적으로 끝남**
Codex의 III-1은 "의과대학 학생이 합격을 위해 사용한다"는 수준에서 멈췄다. 그러나 더 구체적으로 들어가야 한다: 이 학생이 시험 몇 주 전에 이 앱을 여는지, 하루에 몇 분을 쓰는지, 강의록 탭과 기출 탭 중 어느 것을 먼저 여는지, 어떤 차시를 제일 어려워하는지. 사용자 경험 분석이 행동 패턴 수준까지 내려가지 않으면 유인 요소 설계로 이어지기 어렵다.

**[비판] III-3 제안이 개발 공수를 고려하지 않음**
Codex가 제안한 "강의록 강조 텍스트 추출(볼드, 밑줄, 색상)"은 PDF가 스캔본인 경우 기술적으로 불가능에 가깝다. 강의록 PDF 다수가 이미지 기반 스캔임을 이미 gen_study_guide.py 작성 과정에서 경험했을 것이다. 실현 불가능한 제안은 유인 요소가 아니라 기대만 올리는 약속이 된다. 제안 시 기술적 실현 가능성을 함께 평가해야 한다.

**[비판] "Quick Review 탭" 제안 — 현재 문제의 본질을 해결하지 못함**
시험 직전 압축 복습 탭은 ADD-3의 선지 품질 문제와 무관하다. 현재 C- 등급의 핵심 원인은 '새로운 기능 부재'가 아니라 '기존 기능의 콘텐츠 품질 붕괴'다. 새 기능 제안에 앞서 현재 기능의 품질을 끌어올리는 것이 우선이다.
