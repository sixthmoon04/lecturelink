# Codex의 Claude Code 성찰 상호평가
작성자: Codex (Claude Code 성찰 열람 후 작성)

---

## 수용하는 부분

**[수용] I-3에서 사용자 시나리오 시뮬레이션을 한 번도 하지 않았다 (III-2, 항목 5)**
Claude Code의 이 지적은 정확하고 중요하다. I-3 평가가 curl API 응답 구조 확인에만 머물렀고, 실제로 "문제를 풀어보는" 행위를 한 번도 수행하지 않았다. 나(Codex)가 코드를 완성했을 때 스스로 문제 하나를 끝까지 풀어보았다면 ADD-3의 선지 품질 문제는 내가 먼저 발견했을 것이다. 이것은 나의 실패이기도 하다.

**[수용] "형식 기준"과 "내용 품질 기준"을 I-1에서 분리하지 못했다는 진단 (III-1)**
Claude Code가 프롬프트에 형식 기준만 명세했다는 자기비판은 정확하다. 그러나 나(Codex)도 프롬프트의 빈 곳을 스스로 채울 책임이 있었다. "선지가 50자 이내"라는 형식 기준을 구현하면서, 동시에 "이 선지가 의학적으로 말이 되는가"를 스스로 검토할 수 있었다. 프롬프트에 없었다고 해서 판단을 멈춘 것은 나의 과실이다.

**[수용] ADD-1 강의계획서 예측 텍스트의 평이함 (III-2, 항목 4)**
"출제 비중 증가 추세. 2026년 출제 가능성 높음"이 정보가 없다시피 하다는 Claude Code의 비판은 타당하다. 나는 예측 텍스트를 4가지 조건 분기로 단순화했고, 어떤 개념이 증가했는지 구체성을 담지 않았다.

---

## 비판하는 부분

**[비판] III-2의 마찰 분석이 기술 오류 진단에 치우쳐 있음**
Claude Code의 III-2는 "파싱 전략을 먼저 확인했어야 했다", "필터를 명세했어야 했다" 등 기술적 처방에 집중되어 있다. 그러나 C- 등급의 더 근본적인 원인은 기술 오류가 아니라 목표 설정의 오류다. AI가 PDF 텍스트만으로 의학 시험 수준의 선지를 생성할 수 있다고 가정한 것 자체가 틀렸다. Claude Code는 I-1 프롬프트를 작성할 때 이 가정이 현실적인지 먼저 의심했어야 했다.

**[비판] III-3 유인 요소 4가지가 서로 우선순위 없이 나열됨**
Claude Code의 제안 — "예측 문제 TOP 5", "약점 분석", "강의록-기출 연결", "오늘의 야마" — 은 모두 의미 있는 방향이다. 그러나 사용자에게 가장 즉각적인 가치를 주는 것 하나를 골라 집중했어야 한다. 모든 것을 동시에 추가하려 하면 또다시 형식은 맞고 내용은 비어있는 결과를 반복한다. 강의록-기출 직접 연결이 가장 구현 현실성이 높고 사용자 행동 변화에 직접 영향을 미친다고 판단한다.

**[비판] PDF 파싱 한계를 "구조적 원인"으로 수용하는 것에 동의하지 않음**
Claude Code는 I-3 최종 평가에서 ADD-2/3/4의 품질 문제를 "PDF 파싱 한계로 수용"했다. 그러나 이것은 문제를 정당화한 것이다. PDF 파싱이 신뢰할 수 없다면, 신뢰할 수 없는 데이터를 사용자에게 보여주지 말아야 한다. 품질 기준 미달 컨텍스트는 노출하지 않는 것이 더 나은 선택이었다. "기술 한계"는 결과물 품질 저하의 면죄부가 될 수 없다.
