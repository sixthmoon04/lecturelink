## III-1 사용자 경험 시각에서의 성찰

**이 결과물을 누가, 왜, 어떻게 쓰는가?**

이 앱의 사용자는 시험을 앞둔 의과대학 학생이다. 그들이 이 앱을 여는 이유는 단 하나: 시험에 합격하기 위해서다. 그들은 한정된 시간 안에 최대한 많은 기출 문제를 풀고, 강의록의 핵심을 정리하고, '올해 어디서 나올지'를 예측하려 한다.

내가 생성한 코드는 이 사용자 경험의 핵심을 놓쳤다. 사용자가 앱에서 기대하는 것은 '의과대학 시험에서 실제로 나올 수 있는 수준의 문제'이다. 내가 생성한 ADD-3 신규 문제는 "A는 B와 다른 개념이다"라는 형태로 만들어졌는데, 이는 의과대학 시험의 어느 문항과도 닮지 않았다. 내가 PDF 텍스트를 keyword 단위로 쪼개어 선지를 만들 때, 그것이 실제 의학 지식을 묻는 문항이 될 수 있는지를 한 번도 검토하지 않았다.

**핵심 실수:** 나는 "데이터를 처리해서 문제 형식을 맞추는 것"에 집중했고, "이 문제가 실제로 학습에 도움이 되는가"를 코드 로직의 검증 기준으로 삼지 않았다.

## III-2 불편함 재고 — 모든 마찰 요소 점검

1. **gen_3step.py의 선지 생성 로직 근본 결함**
   - 정답 선지: summary의 첫 문장을 그대로 가져왔는데, summary 자체가 PDF에서 추출한 원문이므로 맥락 없이 잘린 문장이 됨
   - 오답 선지: `f"{kw1}는 {kw2}와 다른 개념이다"` 형태는 의학적 판단을 전혀 담지 않음
   - 수정해야 할 것: 정답 선지는 "해당 질환/개념의 특징적 임상 소견"을, 오답은 "같은 질환군의 유사하지만 틀린 내용"을 담아야 한다

2. **gen_study_guide.py의 제목 추출 한계**
   - PDF가 스캔본이거나 한 줄 텍스트 구분이 명확하지 않은 경우, 강의 내용의 첫 문장이 제목으로 선택됨
   - 예: "내과계에서는 호흡기 내과 중환자가 많음" → 교수가 서두에 한 말이 개념 제목이 됨
   - 내가 해야 했던 것: 제목 후보에서 동사 어미(많음, 됩니다, 합니다 등)가 포함된 문장은 제거하는 필터를 추가했어야 했다

3. **gen_minor.py의 lecture_context 품질 필터 부재**
   - OCR 노이즈("배까지증", "태우는연기")가 걸러지지 않고 사용자에게 그대로 노출됨
   - 최소한 한글 자모가 단독으로 나타나거나, 의미 없는 문자열(5자 이하 단어가 전체의 40% 이상)이면 해당 컨텍스트를 제거했어야 함

4. **API 응답 검증 없이 데이터 전달**
   - 생성된 JSON을 app.py가 그대로 프론트엔드에 전달함
   - 최소한 각 문제의 선지가 5개인지, 각 선지 길이가 일정 범위 안에 있는지 런타임 검증을 추가했어야 함

5. **사용자가 신규 생성 문제와 기출 문제를 구분할 수 없음**
   - 3step 탭에서 기출 야마형과 AI 생성 심화/지엽형이 같은 QuestionCard로 표시됨
   - 사용자는 이 문제가 실제 기출인지 AI가 만든 것인지 모른 채로 풀게 됨
   - AI 생성 문제임을 명확히 표시(배지 등)해야 신뢰도를 유지할 수 있다

## III-3 사용자가 원하게 만들 강력한 유인 요소 제안

1. **실제 기출 문제 풀이 경험의 충실한 재현**
   - 현재 선지 클릭 UX는 잘 동작하지만, 정오답 표시 이후 "왜 틀렸는지"를 가르쳐주는 해설이 부족한 문제가 많다
   - 해설이 있는 문제 비율을 높이고, 없는 문제는 명시적으로 "해설 없음"을 표기해 신뢰를 유지해야 한다

2. **강의록 내 개념과 기출 문제의 직접 연결 (cross-reference)**
   - 학습 가이드의 각 개념 키워드를 클릭하면 해당 키워드가 포함된 기출 문제 리스트로 이동
   - "이 개념이 이렇게 출제됐구나"를 즉각 경험하게 해주는 것이 가장 강력한 학습 유인

3. **AI 생성 문제의 품질을 높여 "기출보다 더 어려운 예상 문제" 포지셔닝**
   - 현재의 선지 품질로는 보조 자료로도 사용하기 어렵다
   - 강의록에서 교수가 직접 강조한 문장(볼드, 밑줄, 색상 강조)을 추출해 선지 근거로 사용하면 품질이 올라간다
   - 이 문제를 풀었을 때 "아 이게 나올 수도 있겠다"는 느낌을 줘야 한다

4. **시험 직전 "핵심만 보기" 모드**
   - 강의 가이드와 야마형 문제만 한 화면에 압축해서 보는 Quick Review 탭
   - 시험 당일 아침에 30분 만에 전체를 훑을 수 있는 구성이면 앱을 반드시 열게 된다
