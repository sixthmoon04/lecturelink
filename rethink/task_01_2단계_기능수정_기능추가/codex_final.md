# Codex 최종 성찰 (갱신)
과제: 2단계 기능 수정(FIX-1~4) + 기능 추가(ADD-1~4)
등급: C-
갱신 근거: Claude Code 상호평가서([비판] 3항목) 반영

---

## 최종 성찰 — 핵심 명제

**"실행이 되면 완성"이 아니라 "사용자가 가치를 얻으면 완성"이다.
그리고 유인 요소 제안은 기술 실현 가능성을 먼저 검토한 후 해야 한다.
새 기능은 기존 기능의 품질이 안정된 이후에 논의할 수 있다.**

---

## 최종 성찰 상세

### [원인 1] 스스로 결과물을 사용해보지 않았다
Claude Code의 지적("I-3에서 문제를 한 번이라도 풀어봤다면 즉시 발견했을 것")을 수용한다.
앞으로: 콘텐츠 생성 스크립트(gen_3step.py, gen_study_guide.py 등) 완성 후 결과물 무작위 3개를 콘솔에 전문 출력해 Claude Code가 내용을 확인할 수 있도록 한다. 이 확인 절차 없이 app.py에 데이터를 연결하지 않는다.

### [원인 2] 프롬프트의 빈 곳을 맥락으로 채울 책임이 있었다
프롬프트에 명시되지 않았다고 판단을 멈추는 것은 과실이다. 의과대학 시험 맥락에서 "의학적으로 유효한 선지"는 당연한 요건이었다. 앞으로 도메인 맥락이 명확한 경우 프롬프트의 공백을 맥락 기반으로 스스로 채우고, 그 판단 근거를 콘솔 출력에 명시한다.

### [원인 3] 품질 미달 데이터는 출력에서 제외하는 필터를 코드에 직접 삽입한다
"일단 생성하고 보자"는 태도를 버린다. OCR 노이즈, 동사 어미 제목, 의미 없는 선지 — 생성된 순간 사용자에게 노출되어서는 안 된다는 것을 스스로 판단할 수 있었다. 앞으로 모든 gen_*.py에 품질 필터 레이어를 의무화한다.

### [원인 4] 사용자 경험 분석을 행동 패턴 수준까지 내려가야 한다
Claude Code의 비판("행동 패턴 수준까지 내려가지 않으면 유인 요소 설계로 이어지기 어렵다")을 수용한다. "의과대학 학생이 합격을 위해 사용한다"는 분석은 추상적이다. 앞으로 III-1 작성 시 "시험 몇 주 전 / 하루 몇 분 / 어떤 탭을 먼저 / 어느 차시를 어려워하는지" 수준의 행동 시나리오를 포함한다.

### [원인 5] 유인 요소 제안은 기술 실현 가능성 먼저, 새 기능은 기존 품질 안정 후
Claude Code의 비판 2가지를 모두 수용한다.
- "강의록 강조 텍스트(볼드/밑줄) 추출" 제안은 스캔 PDF 환경에서 기술적으로 불가능하다. 철회한다.
- "Quick Review 탭" 제안은 C- 등급의 핵심 원인(콘텐츠 품질 붕괴)과 무관한 새 기능이다. 철회한다.
앞으로: 유인 요소 제안 전 반드시 현재 기능의 품질 상태를 확인하고, 기존 기능 품질이 안정되지 않은 상태에서 새 기능을 제안하지 않는다.
현 시점 유일한 우선 과제: **기존 콘텐츠 품질 필터 강화 + 강의록-기출 직접 연결**.
