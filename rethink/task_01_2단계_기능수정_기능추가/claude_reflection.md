## III-1 사용자 경험 시각에서의 성찰

**이 결과물을 누가, 왜, 어떻게 쓰는가?**

사용자는 국내 의과대학 학생이다. 본과 시험을 앞두고 호흡기학 2차 기출문제를 반복해서 풀고, 강의록을 정리하며, 교수별 출제 패턴을 분석해 '올해 무엇이 나올지'를 예측하려 한다. 이 앱이 없다면 PDF 파일을 직접 열어 문제를 찾고, 해설을 따로 뒤지며, 강의록과 기출을 수동으로 연결해야 한다. 즉, 이 앱의 핵심 가치는 **'시간 절약 + 시험 합격 확률 상승'** 이다.

**내가 놓친 것:**

나는 I-1 프롬프트를 작성할 때 "기술적으로 구현 가능한 것"에 집중했고, "의과대학 학생이 실제로 이 문제를 풀며 무엇을 느낄지"를 구체적으로 상상하지 않았다. ADD-3 심화/지엽형 신규 문제를 설계할 때 '5지선다 형식을 맞춘다'는 형식 기준만 제시했고, '의학적으로 의미 있는 선지인가'라는 내용 기준을 프롬프트에 명시하지 않았다. 결과적으로 Codex는 형식은 맞췄지만 내용이 비어 있는 문제를 생성했다. 사용자가 "중환자는 중증 및 인지와 관련된 핵심 개념이다"라는 선지를 보는 순간, 이 앱에 대한 신뢰가 무너진다.

**핵심 실수:** 프롬프트에서 '형식 기준'과 '내용 품질 기준'을 분리하지 못했다.

## III-2 불편함 재고 — 모든 마찰 요소 점검

1. **ADD-3 선지 품질 (가장 심각)**
   - "A는 B와 다른 개념이다" 형태 선지는 의학 시험 선지가 아니다
   - 사용자가 문제를 풀다가 이 선지를 보면 앱을 닫을 것이다
   - 원인: I-1 프롬프트에서 '오답 선지 생성 로직'을 단순한 키워드 조합으로 명세했고, 이것이 의학적 오답과 무관한 문장을 만들어냈다
   - 내가 해야 했던 것: "오답은 동일 질환군에서 흔히 혼동되는 임상 사실이어야 한다"는 의학 콘텐츠 기준을 명시해야 했다

2. **ADD-2 개념단위 제목 품질**
   - "내과계에서는 호흡기 내과 중환자가 많음"은 개념 제목이 아니라 강의록 첫 문장이다
   - 학습자가 개념 체계를 한눈에 파악할 수 없다
   - 원인: PDF 텍스트 파싱에서 '제목 추출 규칙'이 너무 느슨했다. 나는 "10자 이하 줄"을 제목 후보로 제시했으나, 강의록 PDF 다수가 스캔본이라 이 규칙이 작동하지 않는다
   - 내가 해야 했던 것: 강의록 PDF의 실제 구조(스캔 vs 텍스트, 목차 존재 여부)를 먼저 확인하고 파싱 전략을 달리 세웠어야 했다

3. **ADD-4 lecture_context OCR 노이즈**
   - "흡연 담배연기 배까지증 형태와도관 태우는연기"는 사용자에게 노출되면 안 되는 데이터다
   - 신뢰성을 해치는 정도가 아니라, 실제로 잘못된 정보를 제공하는 것이다
   - 내가 해야 했던 것: 출력 전 최소 품질 필터(한글 자모 단독 등장, 길이 5자 이하 단어 비율 30% 초과 시 해당 context 제거)를 프롬프트에 명세했어야 했다

4. **ADD-1 강의계획서 예측 텍스트의 평이함**
   - "출제 비중 증가 추세. 2026년 출제 가능성 높음"은 정보가 없다시피 하다
   - 어느 주제가 증가했는지, 어떤 개념을 준비해야 하는지 구체성이 없다
   - 사용자는 이 한 줄을 읽고 "그래서 뭘 공부하라고?"라고 느낄 것이다

5. **검증 부재: 실제 사용자 시나리오를 I-3에서 한 번도 시뮬레이션하지 않았다**
   - I-3에서 API 응답과 JSON 구조만 확인했고, "실제로 차시12를 클릭해 3step을 풀어본" 시뮬레이션을 하지 않았다
   - 사용자 입장에서 한 번이라도 문제를 '풀어봤다면' ADD-3 품질 문제를 1차 평가에서 즉시 발견했을 것이다

## III-3 사용자가 원하게 만들 강력한 유인 요소 제안

1. **"이번 시험에 이게 나올 것 같습니다" 예측 문제 TOP 5**
   - 출제경향 분석 + 강의록 내 강조 비중 + 기출 미출제 영역을 결합해 차시별 예측 5문제를 강조 표시
   - 시험 직전 "이것만 보면 된다" 느낌을 줄 수 있어야 한다

2. **틀린 문제 기반 개인 약점 분석**
   - QuestionCard에서 선지를 클릭할 때 정오답 기록 → 세션 종료 후 "이번 차시 정답률 N%" 표시
   - 차시별 약점이 시각화되면 사용자가 다음에 또 열어야 할 이유가 생긴다

3. **강의록 → 기출 즉시 연결**
   - 강의록 탭에서 특정 개념 클릭 시 "이 개념이 포함된 기출 문제 N개" 즉시 이동
   - 강의록을 읽다가 "이게 어떻게 출제됐지?" 를 바로 해결해 주는 것이 핵심 유인

4. **"오늘의 야마" 랜덤 문제 3개**
   - 앱 진입 시 차시 무관하게 과거 기출 중 정답률 낮은(해설 없는) 문제 3개 랜덤 표시
   - 매일 열어야 할 이유를 만들어 재방문 유도
