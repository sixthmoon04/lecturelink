# Claude Code 최종 성찰 (갱신)
과제: 2단계 기능 수정(FIX-1~4) + 기능 추가(ADD-1~4)
등급: C-
갱신 근거: Codex 상호평가서([비판] 3항목) 반영

---

## 최종 성찰 — 핵심 명제

**C- 등급의 뿌리는 기술 오류가 아니라 목표 설정의 오류다.
"AI가 PDF 텍스트만으로 의학 시험 수준 선지를 생성할 수 있다"는 가정을
I-1 단계에서 한 번도 의심하지 않고 프롬프트에 전제로 박아넣었다.**

---

## 최종 성찰 상세

### [원인 1] 가정(Assumption) 검증 없이 프롬프트 작성 — 핵심 실수
Codex의 비판("기술 오류가 아니라 목표 설정의 오류")을 전적으로 수용한다.
내가 III-2에서 "파싱 전략을 먼저 확인했어야 했다", "필터를 명세했어야 했다"고 반성한 것은 모두 구현 수준의 처방이었다. 그러나 진짜 문제는 그 이전 단계에 있었다: AI가 맥락 없는 PDF 원문 조각에서 의학적으로 유효한 선지를 만들 수 있는가?라는 질문을 I-1에서 먼저 해야 했다.
앞으로 I-1 프롬프트 작성 전, 핵심 가정을 명시적으로 열거하고 각 가정의 현실적 성립 여부를 점검하는 단계를 반드시 거친다.

### [원인 2] I-3 평가에서 사용자 시나리오 시뮬레이션 부재
API 응답 구조 확인 = 기능 검증이 아니다. Codex도 이 지적을 수용했다.
앞으로: 각 ADD 기능 평가 시 사용자가 실제로 거치는 최소 1개 경로를 직접 시뮬레이션하고, 그 결과를 평가 기준에 포함한다. 특히 콘텐츠 생성 기능은 생성된 결과물 3개를 전문 열람 후 평가를 시작한다.

### [원인 3] "기술 한계"를 품질 저하의 근거로 사용한 것
I-3에서 OCR 노이즈, 제목 품질 문제를 "PDF 파싱 한계로 수용"한 것은 오판이었다.
기술 한계는 품질 저하의 면죄부가 아니다. 신뢰할 수 없는 데이터는 노출하지 않는 것이 노출하는 것보다 항상 낫다.
앞으로: 품질 기준 미달 데이터는 "데이터 없음"으로 처리. I-3 평가 기준에 "노출 불가 데이터 0건"을 명시한다.

### [원인 4] III-3 유인 요소를 우선순위 없이 나열한 것 — 위계 재정립
Codex의 비판("하나를 골라 집중했어야 한다")을 부분 수용한다.
단, 나열 자체가 문제가 아니라 위계 없는 나열이 문제였다. 유인 요소들 간 명확한 우선순위를 책정하되, 모든 요소를 폐기하지 않는다.

**유인 요소 우선순위 위계**

| 순위 | 유인 요소 | 근거 |
|------|----------|------|
| **1순위** | 강의록 개념 키워드 → 해당 기출 문제 즉시 이동 | 구현 공수 최소(기존 keyword 필터 응용). 사용자가 강의를 읽다가 "이게 어떻게 출제됐지?"를 즉시 해결. 학습 흐름을 끊지 않고 기출과 연결 — 현재 앱에 없는 가장 직접적인 학습 가속 |
| **2순위** | "이번 시험에 이게 나올 것 같습니다" 예측 문제 TOP 5 | 시험 불안을 가장 직접 해소. 단, ADD-3 콘텐츠 품질 문제가 선결되어야 함. 품질 안정 전에는 기출 야마형으로 대체 표시 |
| **3순위** | 틀린 문제 기반 개인 약점 분석 | 재방문 유인 + 개인화 효과가 크나, 현재 앱이 무상태(stateless)여서 세션 기록 저장 구조 추가가 전제됨. 구현 공수 높음 |
| **4순위** | "오늘의 야마" 랜덤 문제 3개 | 일일 방문 습관 형성에 유효하나, 1~3순위 기능 없이 단독으로 작동하면 깊이 없는 게임화로 전락. 상위 순위 구현 후 도입 |

앞으로: 유인 요소 제안 시 이 위계 순서로 구현 우선순위를 결정한다. 상위 순위가 안정되지 않은 상태에서 하위 순위를 먼저 구현하지 않는다.
